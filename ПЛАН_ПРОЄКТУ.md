## Контекст

Поточна система — VBA-макроси в Excel (.xlsm).  
Обмеження: один користувач, тільки Windows+Excel, складне оновлення, немає аудиту.  
Мета: перенести функціональність у веб-додаток зі збереженням бізнес-логіки.

## Технологічний стек

| Шар | Технологія |
|-----|-----------|
| Frontend | Vue 3 + Vite, Composition API (`<script setup>`) |
| Стан | Pinia: `stores/auth.js` |
| Роутинг | Vue Router 4, history mode |
| HTTP | Axios з JWT interceptor (`api/http.js`) |
| Backend | Python 3.12 + FastAPI 0.111 |
| База даних | PostgreSQL 16 |
| ORM | SQLAlchemy 2.0 (declarative, синхронний) |
| Міграції | Alembic |
| Інфраструктура | Docker + docker-compose (без `version:`) |
| Reverse proxy | Nginx у frontend-контейнері (проксіює `/api/` → backend:8000) |
| XLSX-генерація | openpyxl (Фаза 5) |
| PDF-генерація | WeasyPrint + Jinja2 HTML-шаблони (Фаза 6) |

**Важливі пакети:**
- `bcrypt==4.0.1` — pinned! passlib 1.7.4 несумісний з bcrypt ≥ 5.x
- `passlib[bcrypt]==1.7.4`

## Запуск

```bash
# Перший раз (або після зміни коду)
docker compose build && docker compose up -d

# Тільки frontend/backend
docker compose build frontend && docker compose up -d frontend

# Логи
docker compose logs -f backend
docker compose logs -f frontend
```

**URL:** http://localhost:3000  
**Дані для входу:** `admin` / `admin123` (перший запуск — автоствоюється)

**Multi-project server:** backend не публікується назовні. Frontend на `FRONTEND_PORT` (default 3000), зовнішній nginx проксіює до `localhost:3000`.

## Бізнес-логіка UI

### Тип операції (таби Переміщень)
Визначається з qty_in/qty_out — не з переліку "внутрішніх" підрозділів:
- `qty_in > 0` і `qty_out = 0` → **Надходження**
- `qty_out > 0` і `qty_in = 0` → **Вибуття**
- обидва > 0 → **Переміщення**

### Серійне vs несерійне майно
- Серійне: `serial_number` заповнений, `quantity = 1`
- Несерійне: `serial_number = null`, `quantity = N`
- Перемикач у формі фіксує qty=1 для серійних

### Автозаповнення в формі Переміщень
При введенні `item_card_num` → автозаповнює `item_name`, `unit_of_measure`, `nomenclature_code`, `price` з довідника майна (client-side, items вже завантажені)

### Документи майна (items)
`_sync_documents(db, item_id, docs)` в `routers/items.py`:
1. Видаляє всі `item_documents` → `asset_documents` для цього item
2. Створює нові з payload.documents
Викликається при create і update (якщо documents != None)

### Офіційне/неофіційне майно
`is_official BOOLEAN DEFAULT TRUE`. `FALSE` = волонтерське.
- Відображається як badge "волонтерське" у таблиці і картці
- Офіційні звіти (д14, Додаток 47) фільтрують лише `is_official=TRUE`

## Відомі технічні особливості

| Проблема | Рішення |
|----------|---------|
| passlib + bcrypt≥5 → ValueError | pin `bcrypt==4.0.1` в requirements.txt |
| Alembic має бігти до старту uvicorn | `entrypoint.sh`: `alembic upgrade head && exec uvicorn ...` |
| Docker не зберігає exec bit | `ENTRYPOINT ["/bin/sh", "/app/entrypoint.sh"]` — shell явно |
| Backend не має бути публічним | Видалено `ports:` у backend-сервісі в docker-compose.yml |
| Frontend port гнучкий | `"${FRONTEND_PORT:-3000}:80"` в docker-compose + `.env` |
| ItemListRead vs ItemRead | List endpoint → легка схема (без docs); detail → повна |
